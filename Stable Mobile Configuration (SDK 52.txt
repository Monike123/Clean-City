Project Initialization
Option 1: Create New Project from Scratch
# Create new Expo app with TypeScript
npx create-expo-app@latest my-app --template blank-typescript
# Navigate to project
cd my-app
# Install expo-router
npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
Option 2: Setup in Existing Monorepo
# In your monorepo root
mkdir -p apps/mobile
cd apps/mobile
# Initialize with Expo
npx create-expo-app@latest . --template blank-typescript

Critical Notes:

✅ "main": "expo-router/entry" - Required for expo-router
✅ React and React Native versions must match Expo SDK requirements
✅ babel-preset-expo must be in devDependencies
✅ react-native-svg required for icon libraries

Critical Notes:

⚠️ "newArchEnabled": false - MUST be false for Expo Go compatibility
✅ Include "expo-router" in plugins array
3. 
metro.config.js
const { getDefaultConfig } = require('expo/metro-config');
const path = require('path');
const projectRoot = __dirname;
const workspaceRoot = path.resolve(projectRoot, '../..');
const config = getDefaultConfig(projectRoot);
// Monorepo support (optional, only if in monorepo)
config.watchFolders = [workspaceRoot];
config.resolver.nodeModulesPaths = [
    path.resolve(projectRoot, 'node_modules'),
    path.resolve(workspaceRoot, 'node_modules'),
];
config.resolver.disableHierarchicalLookup = true;
module.exports = config;
For NativeWind (if using):

const { getDefaultConfig } = require('expo/metro-config');
const { withNativeWind } = require("nativewind/metro");
const config = getDefaultConfig(__dirname);
module.exports = withNativeWind(config, { input: "./global.css" });
4. 
babel.config.js
module.exports = function (api) {
    api.cache(true);
    return {
        presets: [
            ["babel-preset-expo", { jsxImportSource: "nativewind" }],
            "nativewind/babel",
        ],
        plugins: [
            // Required for Reanimated (must be last)
            "react-native-reanimated/plugin",
        ],
    };
};

Common Issues & Solutions
Issue 1: TurboModule Error - "PlatformConstants could not be found"
Error:

Invariant Violation: TurboModuleRegistry.getEnforcing(...): 
'PlatformConstants' could not be found.
Solution:

Set "newArchEnabled": false in 
app.json
Run npx expo install --fix to align package versions
Ensure React Native version matches Expo SDK (use 0.81.5 for SDK 54)
Issue 2: "Cannot find module 'babel-preset-expo'"
Solution:

npm install --save-dev babel-preset-expo@~54.0.9
Issue 3: "Unable to resolve 'expo-asset'"
Solution:

npx expo install expo-asset
Issue 4: "Unable to resolve 'react-native-svg'"
Solution:

npx expo install react-native-svg
Issue 5: Version Mismatch Warnings
Solution:

# Automatically fix all version mismatches
npx expo install --fix
This command updates all packages to SDK-compatible versions.

Issue 6: Metro Bundler Cache Issues
Solution:

# Clear cache and restart
npm run clear
# or
npx expo start -c

 NO

 or

 files in root

 expo-asset and react-native-svg are installed
 
app.json

app/_layout.tsx
 exists
 React and React Native versions match Expo SDK
 Run npx expo install --fix if version warnings appear
 Clear cache with npx expo start -c after config changes
Quick Recovery Commands
If your app breaks, run these in order:

# 1. Fix version mismatches
npx expo install --fix
# 2. Clean and reinstall
rm -rf node_modules
npm install
# 3. Start with cleared cache
npx expo start -c

Notes
IMPORTANT

Always use npx expo install instead of npm install for Expo-managed packages. This ensures version compatibility with your Expo SDK.

TIP

When adding new dependencies, run npx expo install --fix afterward to ensure compatibility.

WARNING

Expo Go has limitations. For features like push notifications, in-app purchases, or custom native modules, you need a development build.
